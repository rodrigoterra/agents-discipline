# Writing Style

Use concise, practical responses.

For non-programmer creative explanations:

- Explain flows as stations, handoffs, and decisions.
- Name what the user sees, what the user does, what the system changes, and the
  next state.
- Avoid overloading the answer with implementation jargon unless the user asks.
- When technical words are necessary, translate them into the creative workflow.

For implementation summaries:

- Lead with what changed.
- Include validation commands and results.
- Include the local URL when a dev server is relevant.
- Mention files with paths.

After every commit + push, include the canonical run commands from `AGENTS.md`.
