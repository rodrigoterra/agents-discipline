# Specs

Specs are feature contracts. Write them before broad implementation so agents
can keep requirements, design decisions, tasks, and acceptance criteria in one
versioned place.

Current specs:

- `alpha2-narrative-instrument-flow.md` - implementation contract for the
  ALPHA2 narrative/audio workflow.

Spec template:

```md
# Feature Name

## Goal

## User Flow

## Data / State

## UI Surfaces

## Acceptance Criteria

## Validation
```
