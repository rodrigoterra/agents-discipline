# karpathy-guidelines

Behavioral guidelines based on Andrej Karpathy's observations of common LLM
coding mistakes — resist overcomplication, make surgical changes, surface
assumptions, avoid defensive bloat, and define verifiable success criteria.

Ships a single skill, `karpathy-guidelines`, that auto-triggers when writing,
reviewing, or refactoring code.
